## 1. 引言

### 1.1 编写目的
本文档旨在定义一款跨平台 Markdown 桌面编辑器的功能需求、用户界面及非功能性约束。该编辑器以 [Obsidian](https://obsidian.md) 为参考，强调本地化、插件兼容和轻量高效，目标用户是需要频繁编写、阅读及导出 Markdown 文档的个人及团队。

### 1.2 项目背景
现有 Markdown 编辑器或过于简陋，或附带复杂的知识库、数据库、关系图谱功能，造成资源占用过高。本项目拟开发一款专注文件级 Markdown 编辑的桌面应用，保留 Obsidian 式的编辑体验与插件生态，去除库管理与图谱功能，使软件更轻量、启动更快、跨平台一致。

### 1.3 术语与缩写
- **MD**：Markdown
- **WYSIWYG**：所见即所得渲染
- **Source Mode**：源代码编辑模式
- **Plugin**：插件，与 Obsidian 插件结构兼容
- **Vault**：本项目中指用户打开的文件夹（不包含数据库/图谱概念）

## 2. 总体描述

### 2.1 产品视角
- 本软件为独立的桌面应用程序，可关联 `.md`、`.markdown` 文件。
- 基于本地文件系统工作，无需联网。
- 可通过命令行或双击文件直接打开单个 MD 文件或文件夹。
- 安装时自动建立文件关联（可选）。
- 采用插件架构，安装目录内 `plugins` 文件夹存放插件，启动时自动加载，全局生效。

### 2.2 产品功能
1. 打开/编辑/保存 Markdown 文件。
2. 阅读模式与编辑模式切换；编辑模式支持所见渲染与源代码编辑两种子模式。
3. 大纲（目录）视图，根据标题层级动态生成。
4. 完善右键菜单，支持插入图片、表格、链接、列表等。
5. 支持 Markdown 转 PDF 导出。
6. 兼容 Obsidian 插件，复制即用。
7. 内置帮助文件（Markdown 格式），可在编辑器内查看。
8. 无数据库、无关系图谱功能。
9. 跨平台运行（Windows/macOS/Linux）。
10. Windows 端提供 EXE 安装包。

### 2.3 用户特征
- 熟悉 Markdown 语法的技术写作者、开发者、学生。
- 不需要复杂的知识管理系统，仅需便捷的 MD 编辑与预览。
- 有使用 Obsidian 插件的习惯，希望迁移成本低。

### 2.4 运行环境
- **操作系统**：Windows 10/11，macOS 12+，Ubuntu 20.04+ 等主流 Linux 发行版（图形环境）。
- **依赖**：无需额外安装数据库或特定运行时（除非插件依赖）。
- **权限**：用户对安装目录拥有读写权限（用于放置插件），对文档目录拥有读写权限。

### 2.5 设计约束
- 技术选型优先考虑跨平台 GUI 框架（如 Electron、Tauri、Flutter 等），确保 Windows 可生成单一 EXE 安装包。
- 插件系统必须兼容 Obsidian 插件的 `manifest.json`、`main.js`、`styles.css` 结构。
- 不使用任何嵌入式数据库，不提供图谱视图。
- 软件包大小适中，启动时间小于 3 秒（SSD 环境下）。

## 3. 具体需求

### 3.1 功能需求

#### 3.1.1 文件管理
- **新建文件**：`Ctrl+N` 创建空白 Markdown 文档，自动命名为 `未命名.md`，保存时提示命名。
- **打开文件**：`Ctrl+O` 调用系统文件选择框，支持 `.md`、`.markdown`、`.txt` 等文本格式。
- **打开文件夹**：`Ctrl+Shift+O` 选择文件夹作为工作区，左侧文件列表显示文件夹内所有支持的文件。
- **保存**：`Ctrl+S` 保存当前文档；`Ctrl+Shift+S` 另存为。
- **最近文件**：在菜单中列出最近打开的 10 个文件，点击可直接打开。
- **自动保存**：可配置间隔（默认关闭，防止频繁写盘）。

#### 3.1.2 编辑模式
- **编辑/阅读模式切换**：通过工具栏按钮、快捷键 `Ctrl+E` 或菜单切换。
- **编辑模式子模式**：
  - **实时渲染（所见即所得）**：输入 Markdown 语法后自动渲染为格式化文本，光标位置保留源码。
  - **源代码模式**：纯文本编辑，显示原始 Markdown 标记，无渲染效果，提供语法高亮。
  - 默认子模式可配置，可通过状态栏图标快速切换。
- **编辑功能**：
  - 加粗、斜体、删除线、行内代码、高亮等快捷操作。
  - 支持快捷键（如 `Ctrl+B` 加粗）。
  - 支持 Tab 键缩进、Shift+Tab 减少缩进。
  - 支持自动补全括号、引号、列表等。
  - Undo/Redo 不受模式切换影响。

#### 3.1.3 阅读模式
- 将 Markdown 解析渲染为 HTML 显示。
- 支持代码块语法高亮、表格、图片、LaTeX 数学公式（如有插件支持）、脚注、任务列表。
- 链接点击：内部 `[[wikilink]]` 尝试打开对应文件，外部链接调用系统浏览器。
- 图片支持缩放预览。
- 字体、背景色等可由主题或配置调整。

#### 3.1.4 目录（大纲）列表
- 侧边栏独立面板，可显示/隐藏（快捷键 `Ctrl+Shift+T` 或菜单控制）。
- 实时解析当前文档的标题（H1~H6），显示层级缩进。
- 点击标题跳转到文档对应位置。
- 支持折叠/展开各级标题。
- 大纲视图可跟随滚动高亮当前标题。

#### 3.1.5 右键菜单
- 在编辑区右键弹出上下文菜单，内容随编辑模式、选区变化。
- 必须包含项目：
  - **插入**：图片（本地/网络）、表格（指定行列）、链接、无序列表、有序列表、任务列表、代码块、引用块、分割线、当前日期时间。
  - **格式**：加粗、斜体、删除线、行内代码。
  - **编辑操作**：剪切、复制、粘贴、全选、撤销、重做。
  - **文件操作**：在系统文件管理器中显示当前文件、复制文件路径。
- 阅读模式下右键菜单简化为：复制、全选、打印、查看源代码等。

#### 3.1.6 插件系统
- 插件存放路径：`<安装目录>/plugins/<插件名>/`。
- 兼容 Obsidian 插件结构，要求至少包含 `manifest.json`、`main.js`；`styles.css` 可选。
- 启动时自动扫描 `plugins` 目录，加载所有有效插件，无需用户手动启用。
- 提供插件管理界面：列出已安装插件、版本、作者、描述；可禁用/启用（禁用仅忽略加载，不删除文件）。
- 安装新插件：直接将插件文件夹复制到 `plugins` 下，重启或刷新后生效；也支持在配置界面中点击“打开插件文件夹”。
- 插件不限于编辑功能，可扩展阅读视图、添加命令、侧边栏面板等。
- 因无数据库和图谱功能，部分依赖 Obsidian Vault API 的插件可能无法完全运行，应在帮助文档中说明兼容性限制。

#### 3.1.7 Markdown 转 PDF
- 菜单项：“文件 → 导出为 PDF...”。
- 支持自定义页面大小、边距、字体大小、页眉页脚、代码块样式。
- 导出时可选择是否包含大纲作为书签。
- 导出进度条（大文件时）。
- 生成 PDF 后自动在系统默认应用中打开或在文件夹中显示（可配置）。

#### 3.1.8 帮助说明
- 帮助菜单项：“帮助 → 查看帮助”（`F1`）。
- 帮助内容为软件安装目录下的 `HELP.md` 文件，使用软件自身打开，阅读模式下显示。
- `HELP.md` 包含：基本操作、编辑模式说明、快捷键列表、插件安装方法、常见问题等。
- 发布时预置帮助内容，用户可修改覆盖。

### 3.2 用户界面需求

#### 3.2.1 主窗口布局
默认布局：
- **菜单栏**：顶部
- **工具栏**：菜单栏下方（可隐藏）
- **左侧边栏**：文件列表/大纲视图（可切换，可拖拽调整宽度，可隐藏）
- **编辑/阅读区**：中央占据主要区域
- **右侧边栏**：可选，如插件面板（可隐藏）
- **状态栏**：底部，显示当前编辑模式、行列号、字数统计、光标位置语法类型、自动保存状态、插件状态等

**快捷键**
- 列表显示所有可配置命令及其当前快捷键
- 支持自定义修改、恢复默认、冲突检测
- 支持导入/导出快捷键方案

### 3.3 非功能需求

#### 3.3.1 跨平台一致性
- GUI 框架需保证 Windows、macOS、Linux 视觉效果与交互逻辑一致。
- 文件路径使用操作系统规范，内部统一处理。
- 菜单快捷键根据平台差异调整（如 macOS 使用 `Cmd` 替代 `Ctrl`）。

#### 3.3.2 性能
- 打开 100 KB Markdown 文件及渲染的响应时间不超过 1 秒。
- 100 个以下插件加载时间增加不超过 2 秒。
- 滚动与编辑流畅，无明显卡顿。
- 内存占用基准（无文件打开）低于 150 MB；打开一个典型文件低于 250 MB。

#### 3.3.3 安装与更新
- **Windows**：提供 MSI 或 NSIS 安装包，安装后可选择添加桌面快捷方式、关联 `.md` 文件。安装目录为用户可写（允许放入插件）。
- **macOS**：提供 DMG，拖拽至 `/Applications`。
- **Linux**：提供 AppImage、deb 包或 Flatpak。
- 自动更新检查（可配置关闭），提示下载新版本；Windows 下支持安装包覆盖升级。

#### 3.3.4 安全性
- 插件运行在沙箱或有限权限下，不访问网络权限（除非用户明确授予）。
- 不在未加密情况下存储或传输用户数据（本软件无网络功能，不涉及）。
- 文件操作仅限用户主动打开或保存的目录。

## 4. 附录

### 4.1 Obsidian 插件兼容性说明
本软件插件接口兼容 Obsidian 插件规范。由于不包含 Vault、图谱、数据库等 API，部分依赖 `app.vault`、`app.metadataCache`、`app.workspace` 特有方法的插件可能无法完全正常工作。在插件管理界面中标明“部分兼容”，帮助文档中提供建议。

| 打开文件     | Ctrl+O         | Cmd+O       |

| 斜体         | Ctrl+I         | Cmd+I       |
| 查找         | Ctrl+F         | Cmd+F       |
| 替换         | Ctrl+H         | Cmd+H       |
